Preface: Mastering the 4‑Component Framework

This workshop teaches you a simple but powerful structure for every prompt you write. The 4‑Component Framework helps you define who you are talking to, what you need, how the output should look and the guardrails that shape the result. When you fill in each piece, ambiguity disappears and quality increases.

Context

Specify your role, audience, domain and any inputs you must respect. A precise context steers vocabulary, priorities and tone.

Task

Clarify exactly what must be produced. Use strong verbs (prioritize, synthesize, contrast, justify, quantify, score) and list measurable success criteria so “good” has meaning.

Output Format

Define how the result should look: sections, tables, headings, bullet counts or word/character limits. Clear formats make review faster and enable rubrics.

Constraints & Tone

Set guardrails and voice. Add tone (pragmatic, non‑hype), do/don’t lists, compliance notes and caveats for numbers. Include a short verification step or self‑check to reduce hallucinations.

Quick Mapping Guide

Need Put it in
Role, audience, inputs/sources Context
Deliverables, actions, success criteria Task
Sections, tables, bullets, limits Output Format
Tone, do/don’t, risk, verification Constraints & Tone

When to Adapt

  • Fast exploration: keep Context and Task tight; simplify the Output Format to encourage creativity.
  • High‑stakes or brand‑sensitive work: expand Constraints & Tone, add verification and prefer few‑shot with exemplars.
  • Ambiguous requests: insert a short clarification step in the Task (e.g., “Ask me 3 questions if anything is unclear”).

Frequent Pitfalls & Quick Fixes

  • Vague verbs → replace with rank, score, justify, quantify.
  • No limits → add word counts, bullet counts or deadlines.
  • Hype claims → require caveats and placeholders for evidence.
  • One‑off outputs → add [placeholders] so prompts become reusable.
  • Tone drift → use few‑shot with exemplars.
